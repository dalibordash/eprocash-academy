Bot de señales

El primer módulo del sistema — ya operativo
Operativo

El bot de señales es un sistema de análisis automatizado que vigila cuatro mercados clave —EUR/USD, BTC/USD, WTI (petróleo) y XAU/USD (oro)— de forma continua, y envía una alerta a Telegram en el momento en que detecta una configuración de entrada según las reglas definidas.

No ejecuta órdenes por sí solo: su función es informar, para que el trader (o, en el futuro, el bot de ejecución) decida qué hacer con esa información.

Flujo de trabajo

  • Descarga datos de precio en tiempo real de los 4 activos vía yfinance
  • Aplica las reglas de análisis técnico configuradas (indicadores, patrones)
  • Cuando se cumple una condición de entrada, genera una alerta
  • Envía esa alerta al canal de Telegram configurado, vía la API de BotFather

Stack tecnológico

Python yfinance Telegram Bot API .bat launcher

El sistema corre actualmente desde un script local con un launcher .bat para encenderlo fácilmente desde escritorio. El siguiente paso natural es moverlo a un VPS para que funcione 24/7 sin depender de que el equipo esté encendido (ver módulo "Bot monitor").

  • No necesitas estar mirando gráficos todo el día
  • Cubre 4 mercados distintos simultáneamente
  • Las alertas llegan directo a tu Telegram, sin instalar nada extra
  • Reglas de entrada consistentes, sin sesgo emocional
  • Base sólida para escalar hacia ejecución automática

Para usar las señales (usuario final)

  • Cuenta de Telegram y unirte al canal oficial de Eprocash Academy
  • Conocimientos básicos de gestión de riesgo (ver Centro de Herramientas) antes de operar cualquier señal

Para correr tu propia instancia (avanzado)

  • Python instalado (Windows, Mac o Linux)
  • Librería yfinance y dependencias del proyecto
  • Un bot de Telegram creado vía BotFather con su token

No. Ningún sistema de señales garantiza ganancias. Es una herramienta de apoyo para identificar oportunidades según reglas técnicas, pero el riesgo de mercado siempre existe.

El bot consulta los precios de forma continua durante su ejecución. La frecuencia exacta depende de la configuración activa del script.

Actualmente las señales se envían en tiempo real al canal de Telegram. Un historial estructurado (backtesting visible) está en el roadmap del proyecto.

Arquitectura preparada para integración: este módulo está conectado a Telegram vía BotFather. El siguiente paso de integración es exponer un endpoint propio para mostrar señales en vivo directamente en esta página, sin depender solo de Telegram.

Bot de ejecución automática

El siguiente paso evolutivo del bot de señales
En desarrollo

El bot de ejecución automática toma el mismo análisis que genera el bot de señales, pero en vez de solo avisar, coloca la orden directamente en MetaTrader 5 — sin que el trader tenga que abrir el gráfico y operar manualmente.

Hoy este módulo está conectado a una cuenta demo de XM mientras se completa el proceso de backtesting necesario antes de operar con dinero real.

Flujo de trabajo

  • Recibe la señal generada por el módulo de análisis
  • Se conecta a la cuenta de trading vía la librería MetaTrader5 de Python
  • Verifica el símbolo correcto — en XM los símbolos llevan el sufijo m (ej. EURUSDm en vez de EURUSD)
  • Envía la orden con su stop loss y take profit calculados

Stack tecnológico

Python MetaTrader5 (librería) Cuenta demo XM

El descubrimiento de la convención de sufijo m en los símbolos de XM (ej. EURUSDm) fue clave para que la conexión funcionara correctamente — un detalle que no siempre está documentado y que cambia según el bróker.

  • Elimina el retraso entre ver la señal y ejecutar la orden
  • Sin errores de digitación manual al colocar órdenes
  • Stop loss y take profit calculados automáticamente
  • Permite operar mientras no estás frente a la pantalla

Para esta etapa (demo)

  • Cuenta demo en un bróker compatible con MetaTrader 5 (actualmente XM)
  • Terminal MT5 instalado y la cuenta conectada

Antes de pasar a cuenta real

  • Completar el proceso de backtesting (próximo hito del proyecto)
  • Validar resultados en demo durante un periodo prolongado
  • Definir reglas claras de gestión de riesgo por operación (ver Centro de Herramientas)

No todavía. Está en cuenta demo mientras se completa el backtesting necesario para validar la estrategia antes de arriesgar capital real.

Actualmente está probado con XM. Otros brókers compatibles con MetaTrader 5 podrían requerir ajustes, especialmente en la nomenclatura de símbolos.

Depende de los resultados del backtesting en curso. No se dará una fecha fija para evitar promesas que no se puedan cumplir — el progreso se comunicará en el canal de Telegram.

Arquitectura preparada para integración: la conexión MT5 ya existe vía Python. El siguiente paso es exponer el estado de la cuenta (operaciones abiertas, balance) a esta página de forma segura, una vez exista backend.

Bot monitor

La pieza que falta para operar sin supervisión constante
Planeado

El bot monitor es el módulo de vigilancia del sistema: su función no es analizar mercado ni ejecutar órdenes, sino asegurarse de que los otros dos bots estén funcionando correctamente, y avisar de inmediato si algo se detiene o falla.

Este módulo todavía está en fase de planeación. Hoy el sistema se enciende manualmente desde un launcher .bat en un equipo local — el bot monitor es lo que permitirá pasar a una operación verdaderamente autónoma en un VPS.

Funcionamiento previsto

  • Verificación periódica de que el proceso del bot de señales sigue corriendo
  • Verificación de la conexión activa a MT5 (cuando el bot de ejecución esté en producción)
  • Si detecta una caída, envía una alerta inmediata por Telegram
  • Registro de uptime y errores para revisión posterior

Stack tecnológico previsto

Python VPS (pendiente de elegir proveedor) Telegram Bot API

Este módulo depende directamente del despliegue en VPS, que es el siguiente hito planeado del proyecto después del backtesting del bot de ejecución.

  • Detecta caídas del sistema sin que tengas que revisarlo manualmente
  • Permite operar 24/7 sin depender de un equipo encendido
  • Da tranquilidad: sabrás si algo falla antes de perder oportunidades
  • Base para escalar a múltiples estrategias monitoreadas a la vez

Lo que falta antes de construirlo

  • Completar el despliegue del bot de señales y ejecución en VPS
  • Elegir proveedor de VPS (criterio: uptime, ubicación cercana al bróker, costo)
  • Definir qué métricas son críticas de monitorear (proceso activo, conexión MT5, latencia)

Una vez construido (usuario final)

  • No requiere nada adicional — las alertas de estado llegarán por el mismo canal de Telegram

Porque monitorear algo requiere que ese algo ya esté corriendo de forma estable en un servidor (VPS). Primero se valida el bot de señales y ejecución; el monitor llega cuando haya algo concreto que vigilar 24/7.

No. Significa que hoy el sistema corre localmente y se enciende manualmente. El bot monitor es lo que se necesita para automatizar esa supervisión cuando el sistema se mueva a un servidor permanente.
